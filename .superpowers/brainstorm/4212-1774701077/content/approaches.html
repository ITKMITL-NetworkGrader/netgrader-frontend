How should the Timeline chart be fixed?

Pick the approach that best fits how instructors use the monitoring tab

A

Date Range Filter + Real Timeline

Add a date range picker (start → end). Backend returns actual date+hour buckets. X-axis shows real dates like "Jan 20 8pm", "Jan 21 9am". Adaptive granularity: hourly if ≤ 3 days, daily if wider range.

Jan 20 2026 → Jan 21 2026
Jan 20 9am · 10am · 11am · 12pm · 1pm · 2pm · 8pm · 9pm · Jan 21 10am · 11am · 12pm · 1pm
✓ Most informative — every bar is unambiguous
± Requires backend change + UI date picker
B

Quick Preset Buttons Only

Replace the generic 24-hour chart with smart preset buttons: "Exam Day", "This Week", "Overtime Window", "All Time". Each preset queries its own date range without manual date picking.

📅 Exam Day 📆 This Week ⏰ Overtime 📊 All Time
Each preset knows the lab's dueDate, availableFrom, availableUntil and sets range automatically
✓ Zero configuration — one click
± No flexibility for custom investigation windows
C

Both: Presets + Custom Range

Quick preset buttons for the common cases, plus a "Custom…" option that reveals a date range picker. Best of both worlds — fast for normal use, flexible for investigation.

📅 Exam Day ⏰ Overtime 📊 All Time ✏️ Custom…
Jan 20 2026 ▾ → Jan 21 2026 ▾ (shown when Custom is active)
✓ Fast default + full flexibility when needed
± More UI complexity, slightly more backend work

My recommendation: Option C — the presets solve 90% of cases instantly (Exam Day / Overtime are exactly what you investigated today), and Custom covers the rest.
